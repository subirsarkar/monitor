This page discusses a few finer points that make the monitor useful.

  • Progress of any action is reported at the top right corner.
  • On user action control panels are momentarily disabled to avoid accidental operations that may confuse the browser.
  • Job Summary Table
    • sort the table rows by clicking on the column headers
    • use quick-search to filter out unwanted entries
    • double-click a gridid to find the job detail
    Note that a long table takes a while to load and customise. Please be patient.
  • Job detail Tab, which in turn contains other tabbed views
    • only basic information about a job is shown by default. You will notice that certain tabs are inactive. Check 'Enable Job detail Tabs' in case you want to know more. uncheck the same to know less.
    • by default only jobs belonging to the DN are shown, check 'Show jobs from my VO(s)' if you want to see jobs from your collaborators. However, note that you won't be able to see private information
    • checking auto-update will keep on updating information about the selected job every 10 minutes.
  • Control, configuration and logging
    • Selection panel mostly useful for the site admis; select jobs belonging to a (a) queue (e.g cms, cdf), (b)local user (e.g cmsprd, babarsgm), (c) subject or certificate DN, (d) Resource Broker / WMS, (e) batch Worder Node, (f) date when certain jobs got submitted / queued<, (g) date when certain jobs started
    • Diagnosis, look for mis-behaving jobs in this tab, (a) Load=0 - the job does not add any load to the node, (b) Load<30 - CPU time < 30% of Real time, (c) Load>=30 - CPU time >= 30% of Real Time, i.e well behaved jobs, (d) CPU=0 - no CPU time being consumed by the job, waiting for data? stuck for some reason?, (e) proxy expired - user proxy for the job expired, stage-out will eventually fail, (f) high mem/vmem jobs
    • various logger areas. When an error occurs you will be automatically directed to the 'Error Logger'. In case you are curious to know the requests that are sent to the server, you may turn on the 'Log Event' option. Turning on 'Log Debug' be best avoided.
    • Configuration panel that contains the checkboxes mentioned above as well as logging options.
  • Different ways of accessing the web-based monitor (Pisa used as an example):
    • https://gridse.sns.it/jobmon/pisa/jobmon.html, the first job in the list is shown.
    • for a single Grid jobid: https://gridse.sns.it/jobmon/pisa/jobmon.html?gridid=ACTUAL_GRID_JOBID
    • a single job from another web page: https://gridse.sns.it/jobmon/pisa/jobinfo.html?gridid=ACTUAL_GRID_JOBID
