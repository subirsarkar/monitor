NAME

BaseTools::MyStore - A simple wrapper over Storable

SYNOPSIS

  use BaseTools::MyStore;
  my $file = q|my.db|;
  my $obj  = BaseTools::MyStore->new({ dbfile => $file });
  $obj->show;

REQUIRES

  Storable
  Data::Dumper
  Carp

INHERITANCE

none.

EXPORTS

none.

DESCRIPTION

BaseTools::MyStore wraps Storable and can manipulate the underlying data structure, usually a hash reference, using add(), remove(), contains(), get() and other related methods.

Public methods

  • new ($attr): object reference

    Class constructor.

        $attr->{dbfile} - the file that holds the underlying Storable data structure.
  • contains ($key): boolean

    Check if a key exists in the underlying data structure.

  • get ($key): $value

    Retrieve the value for a given key. The value should preferably be a scalar but there is not strict rule.

  • add ($key, $value): None

    Add a new key/value pair to the underlying Storable data structure.

  • remove ($key): None

    Remove the entry corresponding to the $key passed as argument from the underlying Storable data structure.

  • save ($params): None

    Save the updated information back to the default file. One can optionally specify a new data structure to be saved. A different file name which should hold the new records can be specified as well.

        $params->{dbfile} - specify in case you want to save the information in a file different from the one specified at construction
    $params->{info}   - you may even opt to store a completely different data structure [rarely needed]
  • show (None): None

    Show the current content of the Storable.

      print Data::Dumper->Dump([$info], [qw/info/]);
  • info ($info): $info

    set/get the underlying Storable data structure. Return the data structure in both cases.
